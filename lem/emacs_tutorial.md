### studying lem with emacs tutorial

macOS では以下の場所に Emacs のチュートリアルの日本語があります。

````sh
$ iconv -f iso-2022-jp /usr/share/emacs/22.1/etc/TUTORIAL.ja > tut-ja.txt
$ cp tut-ja-orig.txt
$ lem tut-ja.txt
````

### カーソル移動
- `C-v` で 1 ページ下に移動
- `M-x` で 1 ページ上に移動
- `C-l` でカーソル行を画面の中央にする

- `C-b` で前の文字
- `C-f` で次の文字
- `C-p` で前の行
- `C-n` で次の行

- `M-f` で次の単語
- `M-b` で前の単語

- `C-a` で行の先頭
- `C-e` で行の終わり

- lem にはない
   `M-a` で文頭
  - `M-e` で文末

- `M-<` でファイルの先頭
- `M->` でファイルの末尾

- `C-u <n>` で繰り返し
  - `C-u` を何回か押すと `n` が増える

- `C-u <n> C-v` で n 行スクロール
- `C-u <n> M-v` で n 行逆スクロール

- lem にはない
  - `C-u 0 C-l` でカーソルをウィンドウの最上部に移動

### エディタがハングした時
- `C-g` (abort)

- 時間がかかっているコマンドの停止
  - これは lem では試せず
- 入力途中のコマンドの終了

### 使用不可コマンド
- この機能は lem にあるのかな
- `C-x C-l` は特にトラップされませんでした
  - ただし使い方もわからず

### ウィンドウ
- `C-x 1` でウィンドウを一つにまとめる

### 挿入と削除
- lem はデフォルトで `C-h` で削除できる（素晴らしい！）

- `C-u <n> <文字>` で繰り返し入力できる
  - `C-u 8 *` など

- `C-h` でカーソルの直前の文字を削除 (delete)
- `C-d` でカーソル位置の文字を削除 (delete)

- `M-C-h` でカーソルの直前の単語を削除 (kill)
- `M-d` でカーソルの後ろの単語を削除 (kill)

- `C-k` でカーソルの位置から行末までを消去 (kill)
- `M-k` でカーソルの位置から文末までを消去 (kill)

- `C-@` から `C-w` までに選択した領域を削除
- `C-@` (mark-set)

- delete したものは yank 不可能
- kill したものは yank 可能

- `C-k` は空行も消去する
- `C-u <2> C-k` は `C-k` を 2 回実行するのとは異なり、2 行削除する

- `C-y` で消去した文字列を再挿入する

- `C-k` を複数行にわたって連続で実行した後に `C-y` を実行すると、削除された行がまとめて再挿入される

- `C-y` した後で `M-y` を実行すると過去にコピーした文字列をペーストできる

- lem ではできない
  - `C-y C-u <n> M-y` でペーストする文字列を n 個前に設定

### 取り消し
- `C-\` で一つ前の操作を取り消し (undo)
- `C-\` を複数回実行すると、その度に前の操作が取り消される
- 文章を変更しない操作は取り消し対象とはならない
  - カーソルの移動など

- `C-_` は操作の再実行 (redo)

### ファイル
- `C-x C-f` はファイルを開く
- `C-x C-s` はファイルを保存する
- 存在しないファイルを開くと、新しいファイルを作成できる

### バッファ
- `C-x C-f` でファイルを開き直すことで、もともと編集していたファイルに戻れる
- `C-x C-b` はバッファのリストを見る
- `C-x b` はバッファを開く
  - 存在しないバッファ名を指定すると新規バッファを作成できる

- *Buffer Menu* はバッファ名の一覧を保持しているバッファ
- *Messages* バッファにはエコーエリアに表示されたメッセージが格納されている

- `C-x s` は編集済みのバッファそれぞれに対し、保存をするかを確認する

#### 補足

- `C-x k` はバッファを閉じる
- `C-x C-b` でバッファの一覧を表示し `d` でバッファを閉じられる

### コマンドの拡張
- `C-x` はコマンドをキーで入力する
- `M-x` はコマンドをコマンド名で入力する

- コマンドはタブで補完できる

- `C-x C-c` は lem を終了する
- lem にはない
  - `C-z` は一時的にエディタを抜ける

- `C-x 1` はウィンドウを 1 つにする

- lem にはない
  - `M-x "replace-string"

- `M-x lem-version` は lem のバージョンを表示する

### 自動セーブ
- 自動セーブされたファイル名には `#` がつく

- lem にはない
  - `M-x recover file`

### エコーエリア
- lem ではキーシークエンスをゆっくり入力してもエコーエリアには反映されない

### モードライン
- モードラインには編集中のファイルに関する情報が表示される
- `*` は編集済み
- `-` は保存済み
- ファイル名
- メジャーモード
- (行数, 文字位置)

- メジャーモードはコマンドになっている
- `M-x fundamental-mode` で fundamental モードに移行できる

- lem にはない
  - text-mode

- lem にはないと思われる
  - メジャーモードのヘルプ

- `C-u C-v` でテキストを上にスクロールさせる

- lem にはない
  - auto-fill-mode

### 検索
- `C-s` はインクリメンタルサーチ
- `C-r` は逆向きのインクリメンタルサーチ

- インクリメンタルサーチの途中で return を押すと検索が終了する
- インクリメンタルサーチの途中で delete を押すと検索文字が 1 文字削除される

- `C-g` は検索を終了させる

### 複数のウィンドウ

- lem にはない
  - `M-C-v` で別のウィンドウをスクロールさせる

- `C-x o` は他のウィンドウにカーソルを移動する
- 一つのファイルを複数のウィンドウで開くことで、カーソル位置を維持したまま編集ができる

- 操作中のウィンドウは selected window と呼ぶ

- lem にはない
  - `C-x 4 C-f` で新しいウィンドウでファイルを開く

### 再帰編集レベル
- lem に再帰編集レベルはあるのかな?
- lem にはない
   - Esc Esc Esc での抜け出し

### もっともっとヘルプ
- lem にはない
  - `C-h` によるヘルプ
  - `M-x help`

- `C-x ?` はコマンドのキーのヘルプ

### もっと多くの機能
- lem には関係のないチャプターでした
